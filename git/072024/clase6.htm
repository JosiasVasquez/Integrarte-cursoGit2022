Sexta sesión curso de control de versiones con git y github:

Temas vistos en la sesión:

  • Cómo aceptar o rechazar una pull request

    Durante la sexta sesión del curso, aprendimos a gestionar pull requests en GitHub. Esto incluye cómo revisarlas, hacer comentarios sobre el código, añadir sugerencias de cambios y finalmente cómo aceptar o rechazar las pull requests. Revisamos la importancia de las revisiones de código en equipo y cómo un buen proceso de revisión puede mejorar la calidad del software. También discutimos las mejores prácticas para comunicar cambios y resolver conflictos de manera eficaz.
